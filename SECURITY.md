# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, contact maintainers privately with:

- A clear summary of the issue
- Impact and affected components
- Reproduction steps or proof of concept
- Suggested mitigation if known

## Response Targets

- Initial acknowledgement: within 7 days
- Triage outcome: as soon as practical based on severity

## Scope

Security reports should focus on:

- Remote code execution vectors
- Unsafe deserialization or untrusted file handling
- Credential or token leakage
- Supply chain or dependency risks
